Note that compared to Vincent's structure, mine is missing end_screen, which is odd
because as the git history shows, I based mine off-of his, so why would I just remove it?
The reason is that I had the idea that end_screen could be a sub-screen to battle like
spellbook is, an overlay so you can still see the final state of the board underneath,
but wether to do that or go for a separate screen is a different type of decision from
directory structure, in my structure an overlay goes in a subfolder and a new screen is
a separate folder, so this decision is sequestered to this readme so the folder structure
is really about showing my idea of structure and not making design decisions
