The idea of the file structure is one we discussed before, but I will state how I concieve of it right now.
The fundamental idea is that the files of a thing go into that thing's folder. Some things are categories
like "screens", "units", "spells", etc. The files that are common to the category go into the folder for
that category, and subordinate things (spellbook pop-up to battle) or things of that category (unit1 is a
unit) go into that folder. Thinking about it in terms of categories might be a trap though, it's just about
ideas, really, like the folder "drawing" is about the user drawing on the screen.
The main point is to structure in terms of ideas, not file types.
